
# Nomos Code of Conduct

## Overview

Nomos is an open infrastructure project focused on building reputation
systems for autonomous agents in the Web4 AI economy. A collaborative
and respectful environment is essential for the success of this project.

This Code of Conduct defines the expectations for all participants in
the Nomos community, including contributors, maintainers, researchers,
developers, and users.

The goal is to ensure that Nomos remains an inclusive, professional, and
technically productive environment.

------------------------------------------------------------------------

# Scope

This Code of Conduct applies to all spaces related to the Nomos project,
including but not limited to:

-   the GitHub repository
-   issue discussions
-   pull request reviews
-   documentation discussions
-   development channels
-   research collaboration environments
-   community communication channels

Participants are expected to follow this Code of Conduct whenever they
represent the Nomos project.

------------------------------------------------------------------------

# Community Principles

The Nomos project is guided by the following principles:

## Professional Collaboration

Contributors should engage in discussions with professionalism and
technical clarity. Productive disagreement is welcome when expressed
respectfully.

## Technical Integrity

All discussions should prioritize correctness, evidence, and
constructive reasoning.

## Openness

The project encourages open research, transparent development processes,
and community collaboration.

## Respect

All participants should be treated with respect regardless of experience
level, background, or technical perspective.

------------------------------------------------------------------------

# Expected Behavior

The following behaviors are expected from all members of the Nomos
community.

## Respectful Communication

Participants should:

-   communicate respectfully and clearly
-   focus on technical arguments rather than personal criticism
-   welcome constructive feedback
-   assume good intentions when engaging with others

## Constructive Contributions

Contributors should:

-   propose improvements thoughtfully
-   provide clear explanations for changes
-   help review the work of others when possible
-   support new contributors entering the project

## Responsible Research and Development

Participants should:

-   maintain transparency in experimental work
-   clearly document assumptions and limitations
-   avoid misleading claims about protocol capabilities

------------------------------------------------------------------------

# Unacceptable Behavior

The following behaviors are not acceptable within the Nomos community.

## Harassment

Harassment of any kind is strictly prohibited. This includes:

-   hostile or abusive language
-   personal attacks
-   intimidation
-   threats

## Discrimination

Discriminatory behavior based on identity, background, or personal
characteristics is not tolerated.

## Disruptive Conduct

Actions that intentionally disrupt collaboration or development
processes are not acceptable.

Examples include:

-   spam or irrelevant comments
-   intentionally misleading technical claims
-   repeated off-topic arguments

## Abuse of Community Resources

Misusing project infrastructure, communication channels, or
collaborative processes is prohibited.

------------------------------------------------------------------------

# Maintaining a Productive Environment

To ensure productive collaboration, participants should:

-   remain focused on the technical goals of the project
-   keep discussions relevant to the issue or proposal being discussed
-   avoid unnecessary escalation of disagreements
-   provide evidence or reasoning when challenging technical ideas

When disagreements occur, they should be handled through structured
technical discussion.

------------------------------------------------------------------------

# Maintainer Responsibilities

Maintainers have additional responsibilities in preserving the health of
the project environment.

Maintainers should:

-   enforce the Code of Conduct consistently
-   remain neutral and objective during disputes
-   guide discussions back to productive outcomes
-   help de-escalate conflicts when they arise

Maintainers may remove content that violates community standards.

------------------------------------------------------------------------

# Enforcement

Violations of this Code of Conduct may result in actions including:

-   warnings
-   removal of comments or content
-   temporary restrictions from participation
-   permanent bans from the repository or community spaces

Enforcement decisions are made by project maintainers.

Severity of enforcement depends on:

-   the nature of the violation
-   repeated behavior
-   impact on the community

------------------------------------------------------------------------

# Reporting Issues

If you experience or witness behavior that violates this Code of
Conduct, please report it to the maintainers.

Reports should include:

-   description of the incident
-   relevant links or references
-   approximate time of occurrence

Reports can be sent to:

contact@nomosweb.org

All reports will be reviewed confidentially.

------------------------------------------------------------------------

# Conflict Resolution

When conflicts arise between contributors, the recommended resolution
process is:

1.  attempt direct and respectful communication
2.  request mediator assistance from maintainers
3.  escalate to maintainers if necessary

The goal is always to resolve conflicts constructively and fairly.

------------------------------------------------------------------------

# Community Growth

Nomos aims to build an ecosystem of researchers, engineers, and builders
working on trust infrastructure for autonomous systems.

To support long-term growth:

-   experienced contributors should support new developers
-   maintainers should encourage knowledge sharing
-   discussions should remain technically focused and welcoming

------------------------------------------------------------------------

# Open Research Environment

Nomos encourages research and experimentation related to:

-   reputation systems
-   behavioral modeling
-   autonomous agent coordination
-   trust infrastructure

Researchers contributing to the project should clearly separate
experimental work from production-ready implementations.

------------------------------------------------------------------------

# Transparency

Transparency is important for maintaining trust in open infrastructure
projects.

Participants should:

-   disclose potential conflicts of interest
-   provide accurate descriptions of contributions
-   avoid misleading technical statements

------------------------------------------------------------------------

# Updates to This Code

The Nomos Code of Conduct may evolve as the project grows.

Updates may be proposed through pull requests or governance discussions.

Major updates should include community feedback and maintainer review.

------------------------------------------------------------------------

# Acknowledgment

By participating in the Nomos project, contributors agree to follow this
Code of Conduct and help maintain a respectful and collaborative
development environment.
